# Asset Index

This file is the authoritative inventory of the SDK contents.

## Top-Level Files

### `README.md`

High-level overview, scope, and intended usage.

## Contracts

### `contracts/service-descriptor.schema.json`

Defines the metadata contract for service discovery and availability reporting.

### `contracts/runtime-metadata.schema.json`

Defines the runtime diagnostics envelope attached to tool results.

### `contracts/result-envelope.schema.json`

Defines the normalized top-level response shape returned to the frontend or persisted in run history.

### `contracts/artifact-manifest.schema.json`

Defines how generated or returned artifacts are indexed for preview, export, and inspection.

## Generated Samples

### `generated-samples/community_variant_browser/`

Checked-in sample scaffold generated by the SDK.

Use this when you want to inspect a real generated output instead of only tokenized templates.

## Scripts

### `scripts/new-workbench-tool.sh`

Generates a scaffold from the templates by replacing token placeholders in file names and file contents.

The script is intentionally conservative:

- it generates files into a separate output directory
- it does not modify host repo files automatically
- it keeps token replacement explicit and inspectable

### `scripts/verify-contracts.py`

Validation helper for the SDK itself.

Behavior:

- always checks that contract and example JSON files parse correctly
- validates the example payloads against the schemas when `jsonschema` is installed

### `scripts/package-release.sh`

Builds a distributable `.zip` from the canonical SDK assets.

Output location:

- `community-workbench-sdk/dist/`

### `scripts/refresh-sample-tool.sh`

Regenerates the checked-in sample tool scaffold under `generated-samples/`.

## Intended Consumption Order

For a new user:

1. `README.md`
2. `docs/START_HERE.md`
3. `docs/ASSET_INDEX.md`
4. `docs/AI_ASSISTANT_GUIDE.md`
5. `contracts/`
6. `scripts/new-workbench-tool.sh`

For an AI assistant:

1. `MANIFEST.json`
2. `docs/AI_ASSISTANT_GUIDE.md`
3. `contracts/`
4. `templates/`
5. generated output
