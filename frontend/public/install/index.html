Install Parthenon Community

Install from the latest tagged GitHub source archive and launch the Python installer.

Native release packages are paused while signed installers are rebuilt. The supported path uses GitHub source code, Docker, and the in-repo installer so every install starts from the same tested code.

Linux, macOS, or Windows inside WSL 2 curl -fsSL https://parthenon.acumenus.net/install.sh | sh
Source-only releases Python installer is source of truth Windows runs through WSL 2

Supported Install

The bootstrap downloads a tagged source release, places it in ~/Parthenon, and opens the browser installer. Use a pinned tag when you need a repeatable deployment.

Latest release

curl -fsSL https://parthenon.acumenus.net/install.sh | sh

Pinned release

curl -fsSL https://parthenon.acumenus.net/install.sh | sh -s -- --version v1.0.6

Terminal installer

curl -fsSL https://parthenon.acumenus.net/install.sh | sh -s -- --cli -- --community

Before You Start

Docker is running

Use Docker Desktop on macOS or Windows, or Docker Engine with the Compose plugin on Linux.

Local resources are ready

Plan for at least 8 GB RAM and 20 GB of available disk for the Community stack.

Windows uses WSL 2

Open an Ubuntu or Debian WSL shell, enable Docker Desktop WSL integration, then run the same command.

What Happens Next

  1. Source checkout The bootstrap uses Git when available, or falls back to the GitHub source archive for the requested release.
  2. Preflight checks Parthenon verifies Docker, Docker Compose, disk space, ports, and operating-system support.
  3. Stack launch Docker Compose starts Parthenon and the Community sidecars, then the installer reports the local URL and admin setup path.

Release Downloads

GitHub releases provide source code archives only. Native zips, .deb, .snap, and package-manager installers return after they are signed, reproducible, and covered by install smoke tests.
