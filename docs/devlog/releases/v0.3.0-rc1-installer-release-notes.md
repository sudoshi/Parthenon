# Parthenon Installer v0.3.0-rc1 — Zero shell prerequisites

*April 26, 2026*

The release-candidate ahead of v0.3.0 final. The promise of v0.3.0 is simple: **a non-technical user can install Parthenon — and configure it as a public HTTPS server — without ever opening a terminal.** v0.2.0 reached "the installer works if Docker is already there." v0.3.0 reaches "the installer installs Docker too."

This is a prerelease. Validation on a fresh machine is the gating step before v0.3.0 final.

---

## What's new in v0.3.0

### One installer, three platforms, no terminal

When the installer detects something missing — Docker, WSL2 on Windows, the Virtual Machine Platform feature, your user not being in the docker group — a **Fix this** button appears next to the failing check. Click it, your OS pops its native auth dialog, and the missing piece installs itself.

The auth dialog is the same one your OS uses for every other privileged action: GNOME's polkit prompt, KDE's polkit prompt, macOS's Touch ID / password sheet, Windows UAC. **The installer never sees or handles your password.**

### Linux

- **Docker auto-install.** Distro-aware: `apt install docker.io docker-compose-v2` on Debian/Ubuntu, `dnf install docker-ce` on Fedora/RHEL after adding the upstream repo. Detects existing Docker — never reinstalls over a working install.
- **Polkit policy + privileged helper.** Ships at `/usr/share/polkit-1/actions/io.acumenus.parthenon.installer.policy` and `/usr/libexec/parthenon-installer-helper`. Single polkit action with `auth_admin_keep` semantics: one password prompt for the entire installer flow.
- **`pkexec` missing? Copy-paste fallback.** On stripped headless servers without polkit, every Fix-this button shows the equivalent shell command with a Copy button instead of failing.

### Windows

- **WSL2 prereq detection + remediation.** Preflight now checks Virtual Machine Platform Windows feature and `wsl --status`. If either is missing, a Fix this button runs `Enable-WindowsOptionalFeature` or `wsl --install` via UAC.
- **Docker Desktop install via winget.** `winget install Docker.DockerDesktop` after WSL2 is ready. Open-source alternative: Rancher Desktop via the same path.
- **Reboot-aware.** WSL2 install + VM Platform enable both require a Windows restart. The installer persists state to `%LOCALAPPDATA%\ParthenonInstaller\state.json` so when you come back, you see a "Welcome back, did you restart?" banner instead of starting over.

### macOS

- **Probe-first runtime detection.** `/Applications/Docker.app` → `colima status` → `/Applications/Rancher Desktop.app`. First match wins — never reinstalls over a working runtime.
- **Default Docker Desktop, alt Colima.** Docker Desktop is what most Mac users expect; we open the official download page and wait for completion. Colima is the open-source / license-clean alternative for orgs that exceed Docker's free-use thresholds (>250 employees / $10M revenue): `brew install colima docker docker-compose`.
- **Apple Silicon optimisation.** Colima starts with `--vm-type=vz --mount-type=virtiofs` on M-series Macs for best performance.

### Server mode (Linux)

- **Step 1 deployment-mode chooser.**
  - Just on this computer (default — http://localhost:8082, no DNS, no certs)
  - Make it reachable from other computers (server install)
- **Caddy + Let's Encrypt, three-line config.** Selecting server install reveals FQDN + ACME email fields. One click runs the full flow: install Caddy from the official Cloudsmith repo, generate a minimum-viable Caddyfile, validate it, install at `/etc/caddy/Caddyfile`, reload Caddy, open UFW ports 80/443. Caddy auto-fetches the Let's Encrypt cert in 30-90 seconds.
- **Caddyfile is intentionally minimal:**
  ```
  {
      email <your-acme-email>
  }

  <your-fqdn> {
      reverse_proxy localhost:8082
  }
  ```
  HTTPS redirect, HSTS, ACME renewal, OCSP stapling — all handled automatically by Caddy. No certbot cron, no apache reload triggers, no mod_md tuning.
- **Apache + Traefik tracked for v0.4.0.** v0.3.0 ships Caddy as the only working reverse-proxy; the radio shows the others labelled "coming in v0.4.0."

## How to install + test

### Linux (Debian/Ubuntu — most thoroughly tested)

```bash
# Prerequisites — usually already met
sudo apt install -y wget

# Download + verify
cd /tmp
wget https://github.com/sudoshi/Parthenon/releases/download/v0.3.0-rc1/Parthenon.Installer_0.1.0_amd64.deb
wget https://github.com/sudoshi/Parthenon/releases/download/v0.3.0-rc1/Parthenon.Installer_0.1.0_amd64.deb.sha256
expected=$(cat Parthenon.Installer_0.1.0_amd64.deb.sha256 | tr -d '\r\n ')
actual=$(sha256sum Parthenon.Installer_0.1.0_amd64.deb | awk '{print $1}')
[ "$expected" = "$actual" ] && echo "SHA-256 OK" || { echo "MISMATCH"; exit 1; }

# Install + launch
sudo apt install -y ./Parthenon.Installer_0.1.0_amd64.deb
# Then launch from your desktop's app menu (search "Parthenon")
# Do NOT launch from a terminal — must inherit GUI launch context (matches a real user)
```

The installer detects no Docker → shows Fix this → click → polkit prompt → Docker installs → preflight green.

### Linux (Fedora / RHEL)

```bash
sudo dnf install ./Parthenon.Installer-0.1.0-1.x86_64.rpm
```

Same flow; helper uses dnf + Docker, Inc.'s upstream yum repo.

### macOS

Download `Parthenon.Installer_0.1.0_universal.dmg`, drag to Applications, launch. The installer handles Docker Desktop install (or offers Colima alternative) via `open` + Homebrew.

### Windows

Download `Parthenon.Installer_0.1.0_x64_en-US.msi`. Click through MSI → launch from Start menu. WSL2 + Docker Desktop install via UAC, with reboot persistence between steps.

### Server mode (any Linux machine with public IP + DNS)

After the regular install completes:
1. Step 1 → "Make it reachable from other computers"
2. Enter FQDN (must already have a DNS A record pointing at the machine's public IP)
3. Enter ACME email
4. Click "Configure server mode"
5. ~60 seconds later, https://your-fqdn loads with a real Let's Encrypt cert

## Known limitations

- **Phase 5 fresh-machine validation pending.** The 8 backend phases all build clean and unit-test green, but a true end-to-end click-through-the-wizard test on a clean Ubuntu 24.04 box (no Docker installed, no Parthenon work in progress) is what gates v0.3.0 final. If you can run that test, please report results in the discussion linked below.
- **Apache + Traefik reverse proxies — v0.4.0.** Currently the radio shows them with "coming in v0.4.0" labels. Picking either returns a "not yet implemented" error. Use Caddy for v0.3.0 server mode.
- **macOS server mode — v0.4.0.** The macOS handlers are wired for desktop installs; server-mode (LaunchDaemon-style Caddy install) is deferred.
- **DNS-01 ACME challenges — v0.4.0.** Currently only HTTP-01 (port 80 reachability required). DNS-01 needs API tokens for the user's DNS provider; that's a v0.4.0 scope decision.
- **Windows MSI bundles 9.6 MB; Linux .deb 6.5 MB; macOS .dmg 31.6 MB.** First run downloads the source-backed bundle (~80 MB) from this same release page.
