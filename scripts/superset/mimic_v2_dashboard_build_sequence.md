## Chart build order

Build and QA charts in this order. The first ten replace the known v1 problem areas and establish the core dashboard.

### Phase 1: KPI and denominator fixes

1. `ICU Admissions`
   - Dataset: `icu_episode_fact`
   - Viz: `big_number_total`
   - Metric: `COUNT(*)`
2. `Unique ICU Patients`
   - Dataset: `icu_episode_fact`
   - Viz: `big_number_total`
   - Metric: `COUNT(DISTINCT subject_id)`
3. `In-Hospital Mortality %`
   - Dataset: `icu_episode_fact`
   - Viz: `big_number_total`
   - Metric: `ROUND(100.0 * SUM(died_in_hospital) / NULLIF(COUNT(*), 0), 1)`
4. `Median ICU LOS`
   - Dataset: `icu_episode_fact`
   - Viz: `big_number_total`
   - Metric: `ROUND(PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY icu_los_days)::numeric, 1)`
5. `7-Day Readmission %`
   - Dataset: `admission_readmission_fact`
   - Viz: `big_number_total`
   - Metric: `ROUND(100.0 * SUM(readmit_7d) / NULLIF(COUNT(*), 0), 1)`
6. `30-Day Readmission %`
   - Dataset: `admission_readmission_fact`
   - Viz: `big_number_total`
   - Metric: `ROUND(100.0 * SUM(readmit_30d) / NULLIF(COUNT(*), 0), 1)`

### Phase 2: Core replacement charts

7. `ICU LOS by Unit`
   - Dataset: `icu_episode_fact`
   - Viz: `box_plot`
   - Group by: `icu_unit`
   - Raw metric column: `icu_los_days`
8. `Diagnosis Burden vs LOS`
   - Dataset: `admission_diagnosis_summary`
   - Viz: `scatter`
   - Entity: `hadm_id`
   - X: `diagnosis_count`
   - Y: `hospital_los_days`
   - Series: `gender`
9. `Mortality by ICU Unit`
   - Dataset: `icu_episode_fact`
   - Viz: `dist_bar`
   - Group by: `icu_unit`
   - Metric: `ROUND(100.0 * SUM(died_in_hospital) / NULLIF(COUNT(*), 0), 1)`
10. `ICU Unit Benchmark Table`
    - Dataset: `icu_episode_fact`
    - Viz: `pivot_table_v2`
    - Group by: `icu_unit`
    - Metrics:
      - `COUNT(*)`
      - `COUNT(DISTINCT subject_id)`
      - `ROUND(AVG(icu_los_days)::numeric, 2)`
      - `ROUND(100.0 * SUM(died_in_hospital) / NULLIF(COUNT(*), 0), 1)`

## QA gates

Do not publish the dashboard until these checks pass:

1. `30-Day Readmission %` equals admission-level logic from `admission_readmission_fact`, not patient-level logic.
2. `ICU LOS by Unit` shows raw stay distributions, not `AVG(icu_los_days)` as the plotted value.
3. `Diagnosis Burden vs LOS` is one row per admission and bubble/entity configuration does not collapse to size `1`.
4. Lab and microbiology charts are labeled as event-weighted or summary-weighted, not patient prevalence.
5. Native filters correctly scope across all participating datasets.

## Import and verification commands

Generate the datasource bundle:

```bash
python3 scripts/superset/generate_mimic_v2_dataset_bundle.py
```

Copy it into the Superset container and import it:

```bash
docker cp scripts/superset/generated/mimic_v2_dataset_bundle.zip acropolis-superset:/tmp/mimic_v2_dataset_bundle.zip
docker exec acropolis-superset sh -lc '/app/.venv/bin/superset import-datasources -p /tmp/mimic_v2_dataset_bundle.zip'
```

Verify the registered datasets:

```bash
python3 - <<'PY'
import requests
base = 'http://127.0.0.1:8089'
s = requests.Session()
login = s.post(
    base + '/api/v1/security/login',
    json={'username': 'admin', 'password': '***', 'provider': 'db', 'refresh': True},
    timeout=20,
)
login.raise_for_status()
token = login.json()['access_token']
headers = {'Authorization': f'Bearer {token}'}
r = s.get(
    base + '/api/v1/dataset/?q=(filters:!((col:schema,opr:eq,value:superset_mimic)),page_size:200)',
    headers=headers,
    timeout=30,
)
r.raise_for_status()
for row in r.json()['result']:
    print(row['id'], row['table_name'])
PY
```
