```markdown
## Deferred during planning/execution
- **vocab GRANT codification** — The manual `GRANT CREATE ON SCHEMA vocab TO parthenon_migrator`

  the ComputePrsModal disables the submit button with a tooltip when endpointName is null. Future
  Phase 17.1: add `/api/v1/cohort-definitions/{id}/prs/compute` for generic dispatch.
- **PRS caching** — Not implemented (RESEARCH Open Question #3). Re-computing a PRS overwrites
  the prior rows per the (score_id, cohort_definition_id, subject_id) PK. For performance-critical
  scenarios, consider a PRS cache keyed similarly to GwasCacheKeyHasher in Phase 17.1+.
- **Multi-source cohorts** — Plan 04's CohortPrsController::index resolves a single source per
  cohort via cohort.source_id. Cohorts with multiple sources (rare in v1) will only show PRS for
  the primary source.
```
