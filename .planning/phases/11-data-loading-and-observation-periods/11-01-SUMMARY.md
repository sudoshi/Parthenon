## Decisions Made
- Used bigint for all concept_id columns when creating new tables (SNOMED codes like 10628511000119107 exceed int32)
- Adapted existing OMOP table schemas: dropped NOT NULL on year_of_birth (some IRSF patients lack DOB), widened drug_source_value to text, altered observation concept_id columns to bigint
- Replaced embedded newlines with spaces and stripped carriage returns during CSV-to-COPY preprocessing (831 measurement rows + drug_exposure rows had multi-line quoted fields)
- Used IRSF sentinel pattern: person_source_value = person_id::text distinguishes IRSF records from existing Synthea/MIMIC data in shared tables

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Existing OMOP tables have restrictive schemas incompatible with IRSF data**
- **Found during:** First load attempt
- **Issue:** person.year_of_birth NOT NULL, drug_exposure.drug_source_value varchar(50), observation.*_concept_id integer (too small for SNOMED)
- **Fix:** Added adaptExistingSchemas() method that ALTERs columns before loading; changed inferColumnType to use bigint for concept_id columns
- **Files modified:** backend/app/Console/Commands/LoadIrsfCommand.php
- **Verification:** All 9 tables loaded successfully after adaptation
- **Committed in:** cb43c80b1

**2. [Rule 3 - Blocking] CSV fields contain embedded newlines breaking PostgreSQL COPY**
- **Found during:** First load attempt (drug_exposure line 177, measurement line 156700)
- **Issue:** 831+ measurement rows and drug_exposure rows have \r and \n inside quoted CSV fields; PostgreSQL COPY interprets these as row terminators
- **Fix:** Added \r/\n stripping in csvToTempFile() and merged observation processing
- **Files modified:** backend/app/Console/Commands/LoadIrsfCommand.php
- **Verification:** All rows loaded successfully; measurement: 370,581 rows, drug_exposure: 41,866 rows
- **Committed in:** cb43c80b1

**3. [Rule 3 - Blocking] Idempotency check fails for shared tables with existing non-IRSF data**
- **Found during:** First load attempt
- **Issue:** Original idempotency check (COUNT > 0) skipped loading into tables that already had Synthea/MIMIC data but no IRSF data
- **Fix:** Implemented IRSF-specific sentinel detection: vocabulary tables check vocabulary_id='IRSF-NHS', clinical tables check person_source_value = person_id::text via JOIN
- **Files modified:** backend/app/Console/Commands/LoadIrsfCommand.php
- **Verification:** Idempotency works correctly -- re-run skips all tables, --fresh drops only IRSF data
- **Committed in:** cb43c80b1
