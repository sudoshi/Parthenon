**Step A — OpenAPI regen:** `./deploy.sh --openapi` ran cleanly. Scribe regenerated `frontend/src/types/api.generated.ts` (gitignored). No committed delta — per RESEARCH.md §A4, the server's `summarize()` projection already flattened JSONB into the same wire shape. Frontend blast radius confirmed ZERO.

**Step B — Frontend build:** `docker compose exec -T node npx tsc --noEmit` → PASS. `npx vite build` → PASS (1.11s, all chunks produced). SC 11 achieved.

**Step C — PANCREAS smoke-gen — DEFERRED:** End-to-end generation on `E4_DM2` (partial coverage_profile — there are no `universal` endpoints in the live data; distribution is 4,532 partial / 629 finland_only / 0 universal) hit three issues, two fixed, one out of scope:

1. **[Fixed] Migration data-shape bug:** `finngen.endpoint_definitions.qualifying_event_spec` was populated with only `expression_json->'source_codes'` from the INSERT-SELECT in `isolate_finngen_schema.php`, losing `resolved_concepts`. Controller reads `$spec['resolved_concepts']` → every endpoint returned 422 "no resolved concepts (likely CONTROL_ONLY)". Fixed via `UPDATE finngen.endpoint_definitions ed SET qualifying_event_spec = s.expression_json FROM finngen.endpoint_expressions_pre_phase13 s WHERE ed.name = s.name` — codified in the follow-up migration below.
2. **[Fixed] NOT NULL constraint bug:** `finngen.endpoint_generations.cohort_definition_id` was left NOT NULL from its pre-move column definition, but per D-07 new generation rows set `cohort_definition_id = null` (using `finngen_endpoint_name` FK instead). Fixed via `ALTER TABLE finngen.endpoint_generations ALTER COLUMN cohort_definition_id DROP NOT NULL`.
3. **[Deferred — out of scope] R worker `cohort_definition_id` requirement:** The Darkstar R worker (`/app/api/finngen/cohort_ops.R`, function `finngen_endpoint_generate_execute`) hard-stops with `stop("endpoint.generate: cohort_definition_id required")` when `cohort_definition_id` is null, and uses the id as the key when writing to the OMOP `{cohort_schema}.cohort` table. This is a real architectural question the 13.1 plan did not resolve — either the R worker must synthesize an id from the run, create an on-demand `app.cohort_definitions` row (which re-couples what 13.1 decoupled), or write to a FinnGen-specific cohort table. Belongs in a follow-up phase. **Smoke-gen blocked here; dev DB migration is otherwise clean.**
