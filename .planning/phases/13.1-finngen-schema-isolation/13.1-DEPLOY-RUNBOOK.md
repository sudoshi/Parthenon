# Phase 13.1 Deploy Runbook (DEV)

> **Scope:** Dev execution only. Production cutover deferred per CONTEXT.md §"Out of scope".
> **Estimated downtime:** < 2 minutes (5,161-row INSERT-SELECT + DELETE on indexed table).
> **Rollback:** the Rollback Procedure below reverses this migration fully — see `IsolationRollbackTest` for round-trip proof.

## Runbook

Run from `/home/smudoshi/Github/Parthenon` on beastmode:

```bash
set -euo pipefail
cd /home/smudoshi/Github/Parthenon

# 1. Enter maintenance — kicks out HTTP producers; /api/v1/finngen/* returns 503.
docker compose exec -T php php artisan down --retry=60 --secret=migrate-13-1

# 2. Pause finngen supervisor — consumers stop claiming new finngen queue jobs.
docker compose exec -T php php artisan horizon:pause-supervisor finngen

# 3. Drain in-flight finngen runs (cap at 120s = 60 iterations × 2s).
for i in $(seq 1 60); do
  ACTIVE=$(docker compose exec -T php php artisan tinker --execute \
    "echo \App\Models\App\FinnGen\Run::whereIn('status', ['queued','running','canceling'])->count();" | tail -1)
  [ "$ACTIVE" = "0" ] && break
  sleep 2
done

# 4. Run the migration (Laravel auto-wraps in PG transaction).
#    NOTE: uses --path= to scope migration to THIS file only, per user safety rule
#    (feedback_never_migrate_force.md: never run unscoped `artisan migrate --force`).
docker compose exec -T php php artisan migrate \
  --path=database/migrations/2026_04_19_000100_isolate_finngen_schema.php \
  --force

# 5. Regenerate OpenAPI types + rebuild frontend.
./deploy.sh --openapi
./deploy.sh --frontend

# 6. Resume finngen supervisor + exit maintenance.
docker compose exec -T php php artisan horizon:continue-supervisor finngen
docker compose exec -T php php artisan up
```

## Post-flight Verification

```bash
# A. Schema + row counts
psql -h localhost -U claude_dev parthenon -c "\dn finngen"
psql -h localhost -U claude_dev parthenon -c "SELECT COUNT(*) FROM finngen.endpoint_definitions"            # expect: 5161
psql -h localhost -U claude_dev parthenon -c "SELECT COUNT(*) FROM app.cohort_definitions WHERE domain='finngen-endpoint'"  # expect: 0
psql -h localhost -U claude_dev parthenon -c "SELECT column_name FROM information_schema.columns WHERE table_schema='app' AND table_name='cohort_definitions' AND column_name='coverage_profile'"  # expect: 0 rows

# B. Grant integrity
psql -h localhost -U claude_dev parthenon -c "SELECT has_schema_privilege('parthenon_app', 'finngen', 'USAGE')"                # expect: t
psql -h localhost -U claude_dev parthenon -c "SELECT has_table_privilege('parthenon_finngen_ro', 'app.users', 'SELECT')"       # expect: f
psql -h localhost -U claude_dev parthenon -c "SELECT has_schema_privilege('parthenon_finngen_rw', 'app', 'CREATE')"            # expect: f

# C. Coverage profile populated (Phase 13 invariant)
psql -h localhost -U claude_dev parthenon -c "SELECT COUNT(*) FROM finngen.endpoint_definitions WHERE coverage_profile IS NULL"  # expect: 0

# D. HTTP smoke — endpoint browser serves via finngen_ro
curl -s -H "Authorization: Bearer $PARTHENON_TOKEN" \
  http://localhost:8082/api/v1/finngen/endpoints?per_page=5 | jq '.data[0] | keys'

# E. PANCREAS smoke-gen (Phase 13 invariant)
RUN=$(curl -s -X POST -H "Authorization: Bearer $PARTHENON_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"source_key":"PANCREAS"}' \
  http://localhost:8082/api/v1/finngen/endpoints/E4_DM2/generate | jq -r '.run_id')
echo "Run ID: $RUN"

# Poll for completion (up to 5 min)
for i in $(seq 1 60); do
  STATUS=$(curl -s -H "Authorization: Bearer $PARTHENON_TOKEN" \
    http://localhost:8082/api/v1/finngen/runs/$RUN | jq -r '.status')
  echo "status=$STATUS"
  [ "$STATUS" = "succeeded" ] && break
  [ "$STATUS" = "failed" ] && { echo "smoke gen FAILED"; exit 1; }
  sleep 5
done
```

## Rollback Procedure (if needed)

```bash
docker compose exec -T php php artisan down --retry=60
docker compose exec -T php php artisan horizon:pause-supervisor finngen
docker compose exec -T php php artisan migrate:rollback \
  --path=database/migrations/2026_04_19_000100_isolate_finngen_schema.php \
  --force
docker compose exec -T php php artisan horizon:continue-supervisor finngen
docker compose exec -T php php artisan up
```

Rollback restores:
- `app.cohort_definitions.coverage_profile` VARCHAR(16) column
- 5,161 rows with `domain='finngen-endpoint'` (author_id = admin@acumenus.net per RESEARCH.md Assumption A5)
- `app.finngen_*` table naming (SET SCHEMA back + RENAME restores `finngen_` prefix)
- `finngen` schema is DROPPED
