## Decisions Made

### Claude's Discretion resolutions (aligned with plan's `<output>` expectations and RESEARCH.md assumptions)

| Decision | Rationale |
|----------|-----------|
| **Rename buildExpressionJson → buildQualifyingEventSpec** | Method name now matches the typed JSONB column `qualifying_event_spec` on finngen.endpoint_definitions; "expression_json" was a CohortDefinition concept. Plan explicitly calls this out. |
| **Remove CohortDefinition::withoutEvents wrapper in importer** | EndpointDefinition has no observer, so the wrapper is vestigial. The bulk Solr reindex is still triggered post-import via `solr:index-cohorts` (the import command invokes it explicitly). |
| **Snapshot SQL source stays app.cohort_definitions WHERE domain='finngen-endpoint'** | Plan literal: "Line 586 snapshot SQL: `INSERT INTO app.finngen_endpoint_expressions_pre_phase13` → `INSERT INTO finngen.endpoint_expressions_pre_phase13`" — the schema qualifier changes, the source query stays. Post-migration the source is empty, so the INSERT is a no-op; acceptable defense-in-depth. |
| **Tags drop 'finngen-endpoint' sentinel** | That sentinel was the domain marker for CohortDefinition rows; EndpointDefinition doesn't need it (schema IS the namespace). `finngen:dfXX` release tag preserved for filtering. |
| **generate() cohort_definition_id: null for new generations** | Post-13.1, FinnGen endpoints are NOT cohort definitions. Historical generations keep dangling references; new ones populate `finngen_endpoint_name` (D-07 FK). The R worker must key on endpoint_name for new runs — downstream reconciliation in a later phase. |
| **'id' field in JSON responses = $row->name (natural PK)** | EndpointDefinition has no surrogate `id`; the frontend expects an `id` key for cache keying. Using the name (which IS the natural PK) preserves the wire shape without adding a surrogate column. |
| **Enum unwrap in summarize/show** | `$row->coverage_bucket instanceof CoverageBucket ? $row->coverage_bucket->value : $row->coverage_bucket` — safely unwraps the Eloquent enum cast to a string for JSON; tolerates raw-string fallback if the cast isn't applied yet. |
| **$fillable 'finngen_endpoint_name' on FinnGenEndpointGeneration** | D-07 adds the column in Plan 02; the importer + controller now populate it. Added to fillable so mass assignment works. |
| **EndpointDefinition generations() HasMany → finngen_endpoint_name FK** | New relationship pattern: endpoint → its generations via the natural PK join (`finngen_endpoint_name` FK pointing at `endpoint_definitions.name`). Future queries can use `$endpoint->generations` cleanly. |

## Deviations from Plan

### Rule 1/2/3 — auto-fixes (3 minor adjustments)

**1. [Rule 2 - critical functionality] FinnGenEndpointGeneration gained `finngen_endpoint_name` fillable + matching write in EndpointBrowserController::generate()**
- **Found during:** Task 1 (model update) + Task 2 (controller write)
- **Issue:** Plan 02 added the `finngen_endpoint_name` column to finngen.endpoint_generations per D-07; without adding it to `$fillable` AND writing to it on new generations, the D-07 FK split wouldn't activate for Phase 13.1+ runs.
- **Fix:** Added 'finngen_endpoint_name' to FinnGenEndpointGeneration `$fillable`; EndpointBrowserController::generate()'s FinnGenEndpointGeneration::updateOrCreate payload now includes `'finngen_endpoint_name' => $row->name`.
- **Files modified:** backend/app/Models/App/FinnGenEndpointGeneration.php, backend/app/Http/Controllers/Api/V1/FinnGen/EndpointBrowserController.php
- **Commits:** 42ce90571, 0bb8a7c95

**2. [Rule 1 - bug / Rule 3 - blocking] EndpointBrowserController::generate() cohort_definition_id param and response — null instead of `(int) $row->id`**
- **Found during:** Task 2
- **Issue:** The original controller read `$row->id` (the CohortDefinition bigint PK) to pass as `cohort_definition_id` to the R worker via Run params. After 13.1, EndpointDefinition has no `id` column (natural TEXT PK is `name`). Using `$row->id` on an EndpointDefinition instance would silently return null (no such attribute), but nothing in PHP 8 guards against it. Leaving the code as-is would cast null → 0, passing `cohort_definition_id=0` to the R worker and corrupting the generation semantics.
- **Fix:** Explicitly pass `'cohort_definition_id' => null` in the Run params, the FinnGenEndpointGeneration update payload, and the 202 response data. Documented in inline comment. R worker must key on `endpoint_name` for new runs (historical runs keep their bigint cohort_definition_id).
- **Files modified:** backend/app/Http/Controllers/Api/V1/FinnGen/EndpointBrowserController.php
- **Commit:** 0bb8a7c95

**3. [Rule 2 - critical functionality] authorId unset in importer processRow**
- **Found during:** Task 2
- **Issue:** The original importer's `CohortDefinition::updateOrCreate` took `author_id => $authorId`. EndpointDefinition has no `author_id` column (FinnGen endpoints are library content, not researcher-authored). PHPStan would flag `$authorId` as unused-argument.
- **Fix:** Explicitly `unset($authorId);` before the EndpointDefinition::updateOrCreate to signal intentional discard. The function signature keeps accepting `authorId` for caller compatibility (ImportEndpointsCommand still passes it). Documented in inline comment.
- **Files modified:** backend/app/Services/FinnGen/FinnGenEndpointImporter.php
- **Commit:** 0bb8a7c95

**Total deviations:** 3 auto-fixes (all Rule 1/2/3 — no Rule 4 architectural checkpoints needed).

**Impact on plan:** Plan executed as designed. The 3 deviations are consequence-of-migration corrections that the plan text implied but didn't spell out (the plan said "update CohortDefinition refs to EndpointDefinition" without listing every field projection; these 3 adjustments close the gap).
