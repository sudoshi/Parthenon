## Deviations from Plan

### Rule 1 Auto-fix — PHPStan strategy correction (Task 3)

**Found during:** Task 3 verification.

**Issue:** The plan (based on RESEARCH §Pitfall 5) prescribed a single 2-line fix: add `/** @var FinnGenConceptResolver&\Mockery\MockInterface $mock */` above `$mock = Mockery::mock(...)` and remove the old narrowing docblock. Claimed to resolve all 4 PHPStan errors.

In practice this did **not** resolve any of the 4 errors. Root cause: Parthenon's `phpstan.neon` sets `treatPhpDocTypesAsCertain: false`, which tells PHPStan to discount PHPDoc-declared types when narrowing expressions. PHPStan sees `$mock->shouldReceive('…')` and, because the runtime signature returns `ExpectationInterface|HigherOrderMessage|Expectation`, picks the narrowest two (`ExpectationInterface|HigherOrderMessage`), losing `Expectation` (the class that defines `andReturnUsing()` and `andReturn()`). The intersection docblock on `$mock` is never trusted and therefore never propagates through the chain.

**Fix:** Kept the intersection docblock (satisfies the plan's stated goal of typing the mock, removes the subtle-bug risk from the old narrowing doc that stripped MockInterface), AND added `@phpstan-ignore method.notFound` inline on each of the 4 `shouldReceive(...)->andReturnUsing/andReturn(...)` chain lines. The first ignore has an explanatory comment; the rest are bare to stay terse.

**Precedent:** The `@phpstan-ignore-line` / `@phpstan-ignore` idiom is already in use in the codebase at:
- `backend/app/Services/Poseidon/PoseidonService.php:29` (catch block for Monolog throw)
- `backend/app/Console/Commands/ImportDesigns.php` (one occurrence)

**Files modified:** `backend/tests/Feature/FinnGen/ImportEndpointsCommandTest.php`
**Commit:** `07eb754b6`
**Alternative considered:** Scoping an ignoreError entry in `phpstan.neon` to `tests/Feature/FinnGen/ImportEndpointsCommandTest.php`. Rejected — modifying the phpstan.neon file exceeds the plan's scope (would be architectural per Rule 4).
