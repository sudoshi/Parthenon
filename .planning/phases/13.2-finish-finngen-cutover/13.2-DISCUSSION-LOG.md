## R Worker ID Strategy

| Option | Description | Selected |
|--------|-------------|----------|
| Use `finngen.endpoint_generations.id` (Recommended) | bigserial id + 100_000_000_000 offset; already tracked per (endpoint, source) pair; no new schema; unique per generation | ✓ |
| Synthesize a deterministic hash id | R worker computes `cohort_definition_id = hash(endpoint_name + source_key)` at fixed offset; no DB round-trip; hash collision risk | |
| Create on-demand `app.cohort_definitions` row | Partially re-couples 13.1's isolation (domain='finngen-generation'); keeps OMOP conventions intact | |
| Separate FinnGen-specific cohort table | `finngen.cohort` keyed by endpoint_name; full isolation; large blast radius (Abby/Analyses/Atlas UNION updates) | |

**User's choice:** Use `finngen.endpoint_generations.id + 100_000_000_000` as the OMOP `cohort_definition_id`.
**Rationale:** Reuses existing infrastructure (bigserial PK on the generations row, already tracked per pair); no new schema; collision-free offset with decades of headroom; deterministic so the existing `overwrite_existing` gate works correctly.
