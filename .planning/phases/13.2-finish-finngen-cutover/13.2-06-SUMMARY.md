### Rule 3 auto-fix — re-include pgsql_testing in $connectionsToTransact (Task 4)

**Found during:** First Pest re-run after Task 1+2 commits showed users_email_unique violations where none existed before.

**Issue:** Laravel's `RefreshDatabase` trait uses `[null]` (= the default connection) when `$connectionsToTransact` is not set as a class property. Setting the property REPLACES that default entirely. My initial `['inpatient_testing', 'finngen_testing']` omitted `pgsql_testing`, so default-connection writes (User::factory) were no longer rolled back between tests.

**Fix:** Changed `$connectionsToTransact` to `['pgsql_testing', 'inpatient_testing', 'finngen_testing']` to preserve the default behavior + add the two testing-variant connections.

**Files modified:** `backend/tests/TestCase.php`

**Commit:** `e6f5cd15b`

### Analytical stop — Rule 4 escalation (62 residual failures)

**Finding:** After all Task 1-4 fixes, 62 tests remain failing. **43 of those (70%) are identical FK violations on `finngen_runs_user_id_fkey`** when a test seeds a User via `pgsql_testing` and then creates a `Run` via `finngen_testing`. The remaining 19 are cascades/assertion mismatches from the same root cause, plus 2 pre-existing (Redis state leak + IsolationRollbackTest data hygiene).

**Root cause:** Laravel creates a distinct PDO per connection. `pgsql_testing` and `finngen_testing` both target the same physical `parthenon_testing` DB, but Laravel's RefreshDatabase wraps them in separate transactions. PostgreSQL's FK trigger on `finngen.runs.user_id → app.users.id` runs within the finngen_testing transaction's snapshot, which does NOT see the uncommitted User row from pgsql_testing's separate transaction.

**Why NOT auto-fixed:** All three viable fixes require changes well beyond 13.2-06's charter:

- **Option A** (shared-PDO override): ~100 LOC test-bootstrap hack to make both connections reuse one PDO. Technically feasible but needs careful review.
- **Option B** (DatabaseTruncation): 30+ test-file updates; 2-3× slower runs; subtle semantic changes.
- **Option C** (commit-fixture-user-outside-RefreshDatabase): Dozens of test setUp() rewrites; rejects per-test isolation.

**Escalation path:** Documented in `13.2-06-PEST-LOG.md §Residual Blocker` for a follow-on plan 13.2-07 (or fold into 13.3). Option A recommended for smallest surface area + safest semantics.

**Plan 13.2-05 (Wave 4) impact:** NONE. Plan 05 runs against DEV `parthenon` where the `finngen` connection routing is correct by default (via the production `'finngen'` connection pointing at DEV parthenon) and the role grants are already applied. parthenon_testing being fully green is NOT a Plan 05 dependency.
